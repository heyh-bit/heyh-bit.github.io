问题现象

用户反馈

  • 请求 /queryDraftGeoJson 接口返回失败
  • 前端提示“加载地图数据异常”
  • 日志无明显错误信息（静默失败）

初步观察

  • 接口调用耗时高达 2.3 秒
  • 最终返回结果为 { "code": 500, "msg": "Internal Error" }
  • 但应用日志中未打印关键堆栈

定位问题

步骤：

  1. 通过 sc/sm 获取类全路径和方法名
  2. 通过 trace 定位到慢方法和异常入口
  3. 通过 watch 实现异常抓取 (方法名字不对, 需要通过 jad 实现查询真正报错的方法名)
  4. 通过 jad 反编译 + 查看继承关系
  5. 通过 watch 实现异常抓取
  6. 通过 watch 定位代码位置
  7. 通过 jad 查看代码（有源码情况下直接看代码）
  8. 通过 watch 实现问题数据 (入参、异常) 排查
  9. 通过 jad/mc/retransform 临时修复上线(跳过)
  10. 验证闭环(跳过)

问题结论

问题排查步骤：

  1. 访问 queryDraftGeoJson 接口, 发现报 500 错误
  2. 通过 sc/sm 获取 VisualizationController 全类名及方法名
  3. 通过 trace 定位 VisualizationController 类中 queryDraftGeoJson 方法
  4. 通过 watch 实现异常抓取, 发现并不是 QueryDraftGeoJsonHandler service 方法报错
  5. 通过 jad 反编译编码, 查看继承关系
  6. 通过 watch 实现异常抓取
  7. 通过 watch 定位代码位置
  8. 通过 jad 查看源代码
  9. 通过 watch 实现问题数据 (入参、异常) 排查

通过以上问题排查步骤, 得到的结论：

报错的位置是 BuildViewDateComponent 类中 convertElement 方法, 报错的原因是空指针 (java.lang.NullPointerException)。因为 skuId=724 的 modelBO 不存在的数据造成的空指针异常。
