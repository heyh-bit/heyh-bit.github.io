git manual

Git 是分布式版本控制系统，用于高效追踪文件变更、协作开发及回溯历史，保障代码安全与团队协作。

学习路线说明：

  1. 基础篇（绿色）

    • 掌握仓库创建、文件跟踪和提交历史查看
    • 学会撤销错误操作和配置忽略文件

  2. 核心篇（蓝色/灰色）

    • 精通分支创建/切换/合并
    • 掌握冲突解决的核心方法

  3. 协作篇（紫色/粉色）

    • 熟练操作远程仓库（克隆/推送/拉取）
    • 掌握 GitHub 协作全流程（Fork → PR → Review）

  4. 高阶篇（棕色）

    • 优化提交历史的 rebase
    • 临时保存修改的 stash
    • 自动化流程的 Git Hooks 和 GitHub Actions

学习资源直通车：

  1. 交互式练习平台
  2. Pro Git 电子书
  3. GitHub 实战实验室

💡 关键提示：
建议按图顺序逐步深入，每个阶段完成一个实战小项目（如：基础篇→个人日记本；协作篇→参与开源文档修正）。遇到卡点时多用 git --help 查看命令文档！

学习路线说明：

  1. 基础操作（绿色）

    • 起点：创建仓库 → 提交变更 → 查看历史 → 撤销操作 → 配置忽略文件
    • 目标：掌握本地版本控制基础

  2. 分支管理（蓝色）

    • 创建/切换分支 → 合并分支 → 解决冲突 → 理解分支策略
    • 目标：实现高效并行开发

  3. 远程协作（紫色）

    • 克隆仓库 → 推送代码 → 拉取更新 → 远程跟踪 → 多远程管理
    • 目标：掌握团队协作基础

  4. GitHub 工作流（红色）

    • Fork 仓库 → 发起 PR → 代码审查 → Issue 跟踪 → 项目管理
    • 目标：精通开源协作模式

  5. 高级技巧（橙色）

    • Rebase优化 → Stash暂存 → Git钩子 → 子模块 → CI/CD自动化
    • 目标：提升工程效率

学习时间建议：

阶段 内容 建议时长 关键产出
基础操作 本地版本控制 4-6小时 本地项目管理能力
分支管理 并行开发 6-8小时 功能分支开发能力
远程协作 团队协作 8-10小时 多人协作项目经验
GitHub 工作流 开源协作 10-15小时 首个合并的 PR
高级技巧 效率提升 持续学习 自动化工作流

🚀 高效学习法：

  1. 每个阶段完成一个实战项目（如：基础操作→个人笔记库；GitHub工作流→修复开源文档）
  2. 使用 Learn Git Branching 可视化工具练习分支操作
  3. 在真实项目中遇到问题再学习高阶技巧，避免过早陷入复杂概念

Git 基础命令

  • Workspace：工作区
  • Index / Stage：暂存区
  • Repository：仓库区（或本地仓库）
  • Remote：远程仓库
